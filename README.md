# 제목: 비트코인 결제와 기존 금융 지급결제의 구조적 차이 비교

## 레포트 내용
# 1 데이터 모델
기존 금융모델의 경우 계정 모델을 차용한다.
사용자들의 상태가 중앙집중화된 서버에서 관리되며 확인을 위해서는 관리 주체를 통해 상태를 쉽게 확인할 수 있다.

가령 A가 B에게 $5 를 보낸다고 한다면 계정 모델의 방식은 다음과 같다.
1 : A의 잔액은 $30이다.
2 : A의 잔액이 0보다 크기 때문에 거래가 진행된다.
3 : A의 잔액은 $25가 되었고, B의 잔액은 $5가 증가한다.

위와 같이, 계정모델은 어떤 상태변화에 대한 최종 결과를 쉽게 확인이 가능하며 (*잔액 조회)
상태 변동 과정이 아니라 최종 결과에 대해 기록하게 된다.

블록체인 중 비트코인의 경우 UTXO 기반 모델을 사용한다.
UTXO 란 Unspent Transaction Outputs 의 약자로, 사용되지 않은 거래 산출물 정도로 해석이 가능하다.

TXO는 블록체인 네트워크에서 '상태 변경의 의도'가 담긴 출력물로 이해할 수 있다.

가령 A가 B에게 5BTC를 보낸다고 할때 UTXO 모델은 아래와 같이 해석할 수 있다.
1 : A가 B에게 5BTC를 보낸다.
2 : 거래가 유효하다면 (A의 BTC보유량, 디지털 서명 등) A의 '상태 변경 의지'가 발생한다.
3 : A가 B에게 5BTC를 전달했다는 UTXO가 생성된다.

계정 모델과의 가장 큰 차이점은,
A와 B의 최종 상태변경이 기록되는 것이 아니라 위와 같은 거래에 대한 결과물을 기록하는 점이다.

# 2 상태의 확인 vs 상태의 추적
금융시스템 관점에서의 결제는 상태 변화를 수반한다.
기존 금융과 블록체인 모두 상태가 변화하고 이를 반영한다 라는 관점에서는 유사하나,
앞서 다룬 모델의 차이로 인해 관심사가 사뭇 달라진다.

기존 금융모델에서의 결제에 대한 관리는
송금자의 잔액, 수취자의 잔액 등 최종 상태가 오류 없이 변경되었는지 확인하는 데에 있다.
이로인해 사용자 또한 비교적 쉽게 거래를 점검할 수 있으며 관리 주체 또한 명확하다보니 책임의 소재 또한
확실한 편이다.

블록체인 모델에서 결제에 대한 관리는 비교적 복잡한 과정을 거친다.
송금자가 수취인에게 결제를 했다는 거래가 발생하며,
해당 거래를 진행하기 위해 필요한 UTXO들의 집합을 조합하여 결과에 대한 새로운 UTXO를 생성한다.

상태 변경이 이루어지는 것은 아니나, 최종 상태를 추적할 수 있는 결과물들(UTXO) 이 생성된다.

# 3 유효거래 전환까지의 난이도
1) 잔액증명
기존 금융의 경우 최종 잔액이 맞게 표시가 된다면, 그 즉시 유효하다고 판단 가능하다.
블록체인의 경우 잔액을 다루지 않고, 이를 구성하는 UTXO들의 집합이 존재한다. 즉 유효성을 확인하기 위해서는
UTXO들을 조합하여 확인하다보니 소요시간이 비교적 오래 걸린다.

2) 승인
기존 금융의 경우 권한을 가진 주체가 승인을 진행하므로, 해당 주체의 상황에 따른 의존도가 높아진다.
물론 대부분의 상황에서는 승인의 절차 및 복잡도가 높지 않으므로 큰 단점은 되지 않는다고 생각한다.

블록체인의 경우 합의 알고리즘에 따라 블록이 네트워크에 등록될 때 비로소 의미있는 것이 되므로,
수수료에 따라 승인 시간의 변수가 존재한다.

3) 결제의 완결
기존 금융의 경우 결제 즉시 성공으로 보이게 되며, 내부 작업 및 책임은 관리 주체가 진행 및 소유한다.
블록체인의 경우 Transaction이 mempool부터 채택되어 채굴, 등록까지 진행되다보니 오랜 시간이 소요된다.

# 4 비교분석
대부분의 사용자의 관심사는 결제의 과정보다는 결과(잔액) 에 있다고 생각한다.
따라서 블록체인이 추구하는 상태의 추적 모델의 경우 범용적인 상황에서는 승인까지의 소요 시간으로 인한 단점이
아직까지는 더 크지 않을까 생각한다.

하지만 상태가 투명하게 공개되고 누구나 확인할 수 있다는 점에서는 검열 저항성이 높고,
기존 금융에서도 승인 절차가 오래 걸렸던 거래 등에 적용된다면 훨씬 더 장점이 많지 않을까 생각이든다.

web3 블록체인의 장점을 잘 활용할 수 있는 상황 및 분야를 발굴하는 것이 중요할 것으로 보인다.
